About Rust

  • Started on July 7th 2010
  • Designed by Graydon Hoare (Twitter: @graydon_pub, Reddit: graydon2) at Mozilla Research (which also created Firefox)
  • Developed by the Rust Project
  • Incorporates aspects of C++, C#, Swift, Haskell, and Scheme among other languages
  • Rust was the most loved language according to Stack Overflow's 2016, 2017, and 2018 Developer Survey
  • In brief: A systems-level programming language that is incredibly fast and prevents segfaults.

What is a Systems-Level Programming Language?

  • Language that is used for writing system software rather than application software
    • System software is designed to operate and control computer hardware and for running application software.
  • More direct access to physical hardware of machine
  • Other examples: C, C++, Pascal, and Go

Advantages

  • Memory Safety Without Garbage Collection
    • Garbage Collection: Form of automatic data management, but takes a significant proportion of processing time and can have an influence on performance.
    • Garbage Collector: a program that automatically removes unwanted data held temporarily in memory during processing.
    • Rust data pointers are converted to C data pointers without the need for additional tagging or data checks.
  • Zero Cost Abstraction
    • There are no run-time or memory costs that come with doing abstraction in Rust.
    • Java has a small cost of abstraction.
    • Usually isn’t relevant but is taken advantage of with Machine Learning.

    • Disadvantages

  • Verbosity
    • In short: A lot more typing.
    • Python
      words = s.split()
    • Rust
      let words: Vec<_> = s.split_whitespace().map(String::from).collect();
    • Usually isn’t relevant but is taken advantage of with Machine Learning.
  • Underdeveloped Community
    • Rust hasn't had enough time to develop an ecosystem of tools and libraries that other languages might have.
      • One package manager
      • Only a couple of pre-1.0 web servers
      • No dedicated IDE
